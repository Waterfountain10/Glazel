CORE FEATURES - BACKLOG
------------------------

TASK: Implement Task Scheduling
Title: Build task scheduler for distributing work to agents
Body:
Currently agents just send heartbeats. Need to implement actual task scheduling logic:
- Create scheduler in internal/scheduler/
- Implement task queue in Redis
- Add logic to assign tasks to available workers
- Handle task retries on failure
- Implement priority queue for urgent tasks

Acceptance Criteria:
- Orchestrator can push tasks to queue
- Workers can pull and execute tasks
- Failed tasks are retried up to N times

Priority: High
Labels: core-feature, backend
Estimate: 3-5 days

TASK: Implement Build Cache
Title: Add distributed build cache for sharing artifacts
Body:
Main feature that makes Glazel useful - caching build artifacts:
- Create cache storage in internal/cache/
- Implement content-addressable storage (hash-based)
- Add cache hit/miss logic
- Store artifacts in Redis or S3/MinIO
- Implement cache eviction policy (LRU)

Acceptance Criteria:
- Can store build artifacts by hash
- Can retrieve artifacts on cache hit
- Cache size is managed/limited

Priority: High
Labels: core-feature, caching
Estimate: 5-7 days

TASK: Build Artifact Storage Layer
Title: Implement storage backend for build artifacts
Body:
Create storage abstraction in internal/storage/:
- Define storage interface (Get, Put, Delete, Exists)
- Implement Redis storage backend
- Implement filesystem storage backend
- Implement S3/MinIO storage backend (optional)
- Add compression for artifacts

Acceptance Criteria:
- Pluggable storage backends
- Can switch between Redis/FS/S3 via config
- Artifacts are compressed to save space

Priority: High
Labels: core-feature, storage
Estimate: 3-4 days

TASK: Add Task Execution Logic to Workers
Title: Workers need to actually execute build tasks
Body:
Workers currently just send heartbeats. Need to:
- Pull tasks from queue
- Execute build commands (shell exec)
- Capture stdout/stderr
- Upload build artifacts on success
- Report status back to orchestrator

Acceptance Criteria:
- Workers can execute arbitrary commands
- Build output is captured
- Artifacts are uploaded to cache
- Status updates sent to Redis

Priority: High
Labels: core-feature, worker
Estimate: 2-3 days

TASK: Implement Build Task API
Title: Create HTTP API for submitting build tasks
Body:
Need API for users to submit builds:
- Add HTTP server to orchestrator
- POST /tasks endpoint to submit new build
- GET /tasks/:id endpoint for status
- GET /artifacts/:hash endpoint to download
- Implement types in internal/api/

Acceptance Criteria:
- Can submit tasks via HTTP
- Can check task status
- Can download cached artifacts

Priority: High
Labels: core-feature, api
